# FASE: ENTREGA DE PROTOTIPOS (Seguimento)

Nas diferentes entregas de prototipos funcionais incluirase unha **instantánea do código do proxecto** no momento da entrega.

## 1. Codificación

Durante a fase de codificación do proxecto elaboraranse varios prototipos, dos que se indicará a **descrición das funcionalidades implementadas**.

- Debe crearse unha **carpeta no repositorio** e subir o código frecuentemente.
- A medida que se vai codificando, aparecerán problemas e será necesario modificar diversos aspectos da fase de deseño.
- A documentación a elaborar debe recoller:
  - Os **problemas atopados**
  - As **solucións adoptadas**

## 2. Prototipos

- Deberase entregar **alomenos dous prototipos** do proxecto.
- Estes servirán como evidencia de que o proxecto está sendo desenvolvido desde cero por parte do alumno/a dentro do prazo oficial establecido.

Os prototipos consistirán en **demos funcionais** do proxecto, adecuadas ao momento de desenvolvemento no que se fixe a entrega, funcionando como:

- Fitos (milestones)
- Ítems entregables de iteración dentro do desenvolvemento

## 3. Probas

- Deben describirse as **probas realizadas**.
- Incluir tamén as **conclusións obtidas**, detallando:
  - Problemas atopados
  - Como foron solucionados

## 4. Cambios sobrevidos no desenvolvemento

No caso de empregar tecnoloxías diferentes ás plantexadas no anteproxecto, debe incluirse:

- **Descrición da motivación** para o cambio
- **Obxectivos** sobre os que se empregaron
- **Alcance do seu uso**

De xeito análogo, calquera cambio respecto ao anteproxecto debe ser **documentado e xustificado**, incluíndo:

- Inclusión ou eliminación de funcionalidades
- Cambios en tipos de usuarios
- Modificacións de tecnoloxías
- Outros aspectos relevantes
